  1. What types of devices does your experiment need to run on?

  2. Do you need to be able to control network functionality below the IP level (SDN, etc.)?

  3. Do experiments need to reload the OS kernel on the device?

  4. Do you need to write code using a specific language or have a substantial amount of pre-existing legacy code you need to use?

  5. How will you get the devices in your testbed?

  6. Based upon your needs, we recommend your testbed use the RepyV2 sandbox. This sandbox provides access to create files, send TCP and UDP traffic, and run a subset of Python on devices. Do your experimenters require other functionality?

  7. Will your testbed be disconnected from the Internet?

  8. Do you want automatic updates or will you update manually?

  9. What types of experimenter tools do you want to include/use?

  10. Is there more than one experimenter on your testbed? If you and your colleagues will share credentials, answer "No".
